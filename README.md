# DigiTech VLP QA вЂ” Jira Integration

Automatically runs a QA audit when an author moves a lesson pack ticket to **"Ready for QA"** in Jira Cloud. Creates a subtask with all findings, severity ratings, and recommended fixes.

## Setup: 3 steps

### Step 1 вЂ” Deploy the webhook service

**Option A: Vercel (recommended, free tier)**

1. Push this folder to a GitHub repo
2. Go to [vercel.com](https://vercel.com) в†’ New Project в†’ import the repo
3. Add environment variables in Vercel dashboard (Settings в†’ Environment Variables):

| Variable | Value |
|---|---|
| `JIRA_BASE_URL` | `https://yourorg.atlassian.net` |
| `JIRA_USER_EMAIL` | service account email |
| `JIRA_API_TOKEN` | Atlassian API token (see below) |
| `ANTHROPIC_API_KEY` | your Claude API key |
| `WEBHOOK_SECRET` | any random 32+ char string |

4. Deploy. Your webhook URL will be: `https://your-project.vercel.app/api/webhook`

**Get your Jira API token:**
в†’ https://id.atlassian.com/manage-api-tokens в†’ Create API token

**Recommended:** create a dedicated service account (e.g. `vlp-qa-bot@yourorg.com`) in Jira.
Give it permission to create issues in your project. Subtasks will be attributed to this account.

---

### Step 2 вЂ” Create the Jira Automation rule

In your Jira project: **Project Settings в†’ Automation в†’ Create rule**

```
TRIGGER
  Issue transitioned
  From status: (any)
  To status:   Ready for QA

CONDITION
  Issue fields condition
  Issue type = Story   (or Task вЂ” whatever you use for lesson packs)

ACTION
  Send web request
  URL:     https://your-project.vercel.app/api/webhook
  Method:  POST
  Headers:
    Content-Type:      application/json
    x-webhook-secret:  (paste your WEBHOOK_SECRET value)
  Body (JSON):
    {
      "issue": {
        "key": "{{issue.key}}"
      }
    }
```

**Save and enable the rule.**

---

### Step 3 вЂ” Configure your Jira project for subtasks

Make sure **Subtask** issue type is enabled in your project:
- Project Settings в†’ Issue types в†’ check "Subtask" is listed

If your project uses a custom QA issue type instead of Subtask, update `issuetype: { name: "Subtask" }` in `api/webhook.js` line ~119.

## File attachment requirements

The automation reads files **attached to the Jira ticket**. Authors should attach:

| File | Format | Required |
|---|---|---|
| Lesson plan | `.docx` | вњ… Yes |
| Slide deck | `.pptx` | Recommended (enables cross-check) |
| Workbook / key resource | `.docx` or `.xlsx` | Recommended |

File roles are detected automatically from filenames following the DigiTech naming convention:
- `Lesson01_...docx` в†’ Lesson Plan
- `SR1_...pptx` or `KeyResource1_...pptx` в†’ Slide Deck
- `KR1_...docx` or `*workbook*.docx` в†’ Key Resource

---

## Triggering manually / re-running

To re-run the audit after fixes:
1. Move the ticket back to **In Progress** (or any non-QA status)
2. Move it back to **Ready for QA**

This fires the automation again and creates a fresh subtask.

## Troubleshooting

**"No .docx / .pptx / .xlsx files found"**
в†’ Author forgot to attach files. Check the comment on the ticket.

**"Could not identify a Lesson Plan document"**
в†’ Filename doesn't match DigiTech convention. Rename to start with `Lesson` or contain `lesson`.

**Subtask not created / automation not firing**
в†’ Check Jira Automation audit log: Project Settings в†’ Automation в†’ (rule) в†’ Audit log
в†’ Check Vercel function logs: Vercel dashboard в†’ your project в†’ Logs

**Claude API error**
в†’ Check `ANTHROPIC_API_KEY` is set correctly in Vercel environment variables.
